## Teoria
¿Qué es XSS?

Cross-Site Scripting (también conocido como XSS) es una vulnerabilidad/falla que combina la aceptación de etiquetas HTML/script como entrada que se procesan en un navegador sin codificación ni sanitización
Cross-Site Scripting (XSS) es el problema de seguridad de aplicaciones web más frecuente y pernicioso

Si bien existe una defensa sencilla y conocida para este ataque, aún existen muchos casos en la web. La cobertura de las soluciones también suele ser un problema. Hablaremos más sobre esta defensa en breve

## Ejercicio 10 

En el ejercicio 10 trata de XSS DOM, este XSS consiste en manipulando el DOM **(Document Object Model)**.
A diferencia del XSS clásico, el servidor no modifica la respuesta
Todo ocurre del lado del cliente mediante **JavaScript**.

![](/assets/images/2025-12-13-Cross-Site-Scripting/xss10.png)


Nos meteremos en inspeccionar ya que este trata de buscar ficheros js, si miramos las pistas vemos que tenemos que buscar el GoatRoute.js

![](/assets/images/2025-12-13-Cross-Site-Scripting/xss4.png)

Dentro de ese archivo en el código vemos test… si miramos bien el código
entendemos que donde pone **'test/:param': testRoute** quiere decir que cualquier url que tenga el patrón test/param activará testRoute

Esto se puede verificar con

![](/assets/images/2025-12-13-Cross-Site-Scripting/xss4.1.png)

![](/assets/images/2025-12-13-Cross-Site-Scripting/xss4.5.png)


Una vez verificado que es vulnerable ya tendremos la ruta que es

## Ejercicio 11
![](/assets/images/2025-12-13-Cross-Site-Scripting/xss11.png)



Una vez encontrada la vulnerabilidad de antes usaremos el **webgoat.customjs.phoneHome()** para realizar la actividad 11

Tendremos que urlencodear el **webgoat.customjs.phoneHome()** con Burrp

```java
<script>webgoat.customjs.phoneHome()<%2Fscript>
```

Una vez hecho lo pondremos en la url y tendremos que ver una respuesta como esta

![](/assets/images/2025-12-13-Cross-Site-Scripting/xssbur11.png)

Para luego mirar la consola y encontrar el número

![](/assets/images/2025-12-13-Cross-Site-Scripting/xss11.1.png)


Esta respuesta significa que JavaScript ejecutó la función que informa el progreso de su tarea al backend de WebGoat. Básicamente webgoat.customjs.phoneHome hara que en la consola nos aparezca un número aleatorio que tendremos que colocar en la respuesta.

![](/assets/images/2025-12-13-Cross-Site-Scripting/xssultimo.png)


## Ejercicio 3 de Cross-Site Scripting (Storage)
![](/assets/images/2025-12-13-Cross-Site-Scripting/xss3storage.png)

En esta práctica consiste en XSS almacenado lo cual implica poder almacenar JavaScript como una carga útil con el webgoat.customjs.phoneHome para resolver este problema básicamente lanzaremos el  <script>webgoat.customjs.phoneHome()</script> en el chat

```java
<script>webgoat.customjs.phoneHome()</script>
```

Una vez se publica el código java en la consola obtendremos la respuesta
